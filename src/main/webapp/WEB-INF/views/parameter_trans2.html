SSM框架中参数传递方法的相关汇总(2)

 

mybatis中的参数处理（传入）

  • 单个参数

    如果sql语句只传入一个参数，那么主需要用 #{参数名} 就可以获得参数值。事实上，参数名可以为任意值，因为这里只有一个参数。

  • 多个参数

    多个参数时，mybatis会做特殊处理，多个参数会被封装成map：

    key是param1,param2....或者0、1、2索引值，value是参数值。

    因此采用param1，param2的方法获得第一、二个参数。

  • 命名参数

    事实上，一般来讲人们还是习惯使用具体的参数名而不是param1、param2这样的参数名，这时可以通过@param("key")明确指定封装map中的key，就可以通过#{key}取出参数值了。

    如：

    mapper:

    public interface HotelMapper {

public Hotel getHotel(@Param("id")Integer i,@Param("price")Integer j);

}

    mapper.xml:

    <select id="getHotel" resultType="com.pjf.mybatis.po.Hotel">
    select id,hotel_name as
    hotelName, hotel_address as hotelAddress, price from hotel
    where
    id=#{id} and price =#{price}
</select>

  • Map

    上面是通过@param("key")指定一个key,那么我们也可以传入一个自己的Map。

    如：

    mapper：

    
public interface HotelMapper {

public Hotel getHotel(Map<String,Integer> map);

}

    mapper.xml:

    <select id="getHotel" resultType="com.pjf.mybatis.po.Hotel">
    select id,hotel_name as
    hotelName, hotel_address as hotelAddress, price from hotel
    where
    id=#{id} and price =#{price}
</select>

    传参：

    //手动封装一个map传入
Map map =new HashMap<>();
map.put("id", 1001);
map.put("price", 997);
Hotel hotel = hotelMapper.getHotel(map);

  • 实体

    对于复杂的结构实体，我们还可以采用传入一个实体类作为参数原型的方式来进行传参，采用#{类中对应的成员名}的方式获得传入实体对象对应成员变量的值。

 

mybatis中的参数处理（传出）

对于传出的resultType，如果它为：

  • 基本类型

    resultType类型为基本类型

  • List类型

    resultType类型为List中元素的类型

  • Map类型

    每一个键对应select的一个列名，对应的值是该列的值

    resultType类型为Map

    使用Map<String,Object>承接传出的结果

 

resultType返回对象的全部字段值，要想返回部分字段值，则需使用resultMap:

先定义一个resultMap实例：

<resultMap id="getStudentRM" type="EStudnet">
  <id property="id" column="ID"/>
  <result property="studentName" column="Name"/>
  <result property="studentAge" column="Age"/>
</resultMap>

其中：

id:用于设置主键字段与领域模型属性的映射关系，此处主键为ID，对应id。

result：用于设置普通字段与自定义实体模型属性的映射关系

然后执行sql语句：

<select id="getStudent" resultMap="getStudentRM">
  SELECT ID, Name, Age
    FROM TStudent
</select>

就可返回自定义实体getStudentRM结构的信息（ID,Name,Age)。
