防火墙权限

服务器自带的防火墙可能会拦截8080等端口的值，因此不能正常访问tomcat的8080端口。

测试8080端口在防火墙是否开启：

firewall-cmd --query-port=8080/tcp

如果没有开启的话，让防火墙开放8080端口:

firewall-cmd --permanent --zone=public --add-port=8080/tcp

然后重启防火墙即可：

systemctl stop firewalld（关闭防火墙）
systemctl start firewalld (开启防火墙)

项目部署

以Intellij IDEA为例，依次选择Build菜单->Build Artifacts->war->Rebuild，然后会在项目target文件夹下生成war包，将该war包放置在服务器tomcat下的webapps路径下（/usr/local/tomcat/webapps）。

配置/usr/local/tomcat/conf/vhost下的localhost.xml:

<Host name="localhost" appBase="webapps" unpackWARs="true" autoDeploy="true">
  <Context path="" docBase="/usr/local/tomcat/webapps/pet" debug="0" reloadable="false" crossContext="true"/>

  <Valve className="org.apache.catalina.valves.AccessLogValve" directory="logs" 
         prefix="localhost_access_log." suffix=".txt" pattern="%h %l %u %t &quot;%r&quot; %s %b" />
</Host>

(将docBase值设为项目war包部署路径)

最后重启tomcat，即完成部署。
