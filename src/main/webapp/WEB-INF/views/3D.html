使用框架

基于 krpano 框架进行3D导览的开发。

krpano viewer 是一个小型的高性能的，适用于交互性的漫游图的展示器。

它可以应用在Flash或HTML5应用中。它是跨平台的。

总体来说，panof、panob、panod、panol、panor、panou 六个方位的图构成了一个全景的场景。每个场景还需要提供一张preview综合图和一张thumb略缩图。

更为具体的一些技术细节

  • Action.xml

    Action.xml是配置工作函数的文件，具体格式如下：

    <action name="xxx">
你的代码
 </action>

    Action里部分参数的定义如下：

    %1 = name of the hotspot to move

    %2 = new scene

    %3 = hlookat startup position in the new scene

    %4 = vlookat startup position in the new scene

    %5 = startup fov in the new scene

    部分函数如下：

    Get() 获取数据

    Set(xx, xx)设置数据

    Copy() 复制

    Loadscene 加载新的场景

    If( condition, then, else) 判断逻辑

  • Layer.xml

    Layer.xml配置通用的界面元素，将对所有scene生效。

    写法如下：

    <layer name="layer_location" keep="true" 

    type="container" width="220" height="80"

        align="middletop" bgcolor="0x333333" 

        bgalpha="0.6"   x="0" y="40">

</layer>

    其中layer可互相嵌套。 name是唯一标识，keep是是否一直显示，type 是容器类型，width是宽度，height是高度，align是摆列方式， bgcolor是背景颜色 bgalpha是透明度 x和y是偏移量。

  • 各个scene配置的xml

    在这里配置各个场景scene，写法如下

    <scene name="" title="" onstart="" thumburl="" 
lat="" lng="" heading="">
    <view hlookat="0" vlookat="0" fovtype="MFOV" fov="120" maxpixelzoom="2.0" fovmin="70" fovmax="140" limitview="auto" />
    <preview url="" />
    <image>
        <cube url="" />
        <cube url="" devices="mobile" />
    </image>
</scene>

    Scene的参数 name是唯一标识，onstart是加载场景时触发的事件，thumburl是缩略图，lat和lng是坐标。

    View是整个视图的标签，preview是缩略图的标签。Image是存放各个角度的图片的标签。Cube则加载不同方位的全景图片。 device设置为mobile是适配手机。
